| controller/src/modules/ | In progress | Renamed controller module-owned product identifiers, environment variables, metric names, Docker resource names, OpenAPI owner fields, and user-facing messages from the previous product name to Local Studio naming. Hardened NVIDIA GPU telemetry parsing so DGX Spark/GB10 `nvidia-smi` N/A memory fields become finite unified-memory totals instead of null GPU rows. Fixed the vLLM launch executable resolver so a managed install is actually used: extracted managed-venv path helpers (`managedVenvPath`/`managedVenvName`/`managedVenvPython`, `isManagedPythonBackend`, `ManagedPythonBackend`) into the leaf module `controller/src/modules/engines/runtimes/managed-venv.ts` to avoid an import cycle with `engine-spec`, and `engine-jobs.ts` now imports them from there. `resolveVllmPythonPath`/`resolveVllmRecipePythonPath` (`runtimes/vllm-python-path.ts`) now also consult `<data_dir>/runtime/venvs/vllm-latest/bin/python`; `getVllmPythonPath` (`process/backend-builder.ts`) forwards `dataDir`; the vLLM spec's `buildCommand` (`specs/vllm-spec.ts`) passes `config.data_dir` through. Fixes "Executable not found in $PATH: vllm" on a fresh box where only the managed venv exists. `buildEnvironment` (`process/process-utilities.ts`) now prepends the resolved venv `bin/` dir to PATH for the spawned backend subprocess, so vLLM's flashinfer MoE JIT kernel can find `ninja` (which lives in the managed venv). Fixes `FileNotFoundError: 'ninja'` on MoE model launches. Converted the four positional `new HttpStatus(status, detail)` throws in `controller/src/modules/engines/routes.ts` (×2, 409 launch-conflict) and `controller/src/modules/proxy/openai-routes.ts` (×2, 400 body/JSON) to the object form `new HttpStatus({ status, detail })` so they construct the new `Schema.TaggedErrorClass`-backed `HttpStatus` from `controller/src/core/errors.ts`. |
| controller/src/core/ | In progress | Migrated `command.ts`, `errors.ts`, and `function-observability.ts` to Effect v4 primitives, following the `*Effect` + `Effect.runPromise`/`runSync` shim convention already in `controller/src/core/async.ts`. `HttpStatus` is now a schema-backed tagged error via `Schema.TaggedErrorClass` (the `4.0.0-beta.90` tagged-error API; `Schema.TaggedError` is not exported yet in this beta) and still extends `Error` with `status`/`detail`, so `isHttpStatus`, `app.onError`, and the observability middleware work unchanged. `command.ts` exposes `runCommandEffect`/`resolveBinaryEffect` (sync, `Effect.sync` + `Effect.runSync`) and `runCommandAsyncEffect` (async spawn + SIGTERM/SIGKILL timeout registered via `Effect.callback` — `Effect.async` is not exported in this beta — settled with `resume(Effect.succeed(result))`), with the existing sync/promise-returning names kept as thin shims. `function-observability.ts` is rebuilt on `Effect.tryPromise` + `Effect.tap`/`Effect.tapError`, adding `observeControllerFunctionEffect` and keeping `observeControllerFunction` as an `Effect.runPromise` shim so all ~30 route call sites are untouched. `tsc --noEmit`, `eslint` on touched files, `jscpd`, `knip`, and `depcheck` are green; runtime smoke tests confirm parity for command output, `HttpStatus` shape, and observability recording on success/failure. |
| shared/src/ | Not changed this turn | No migration-target changes made in this domain. |
| cli/src/ | In progress | Renamed CLI API defaults, config keys, render copy, and headless/input environment variable references from the previous product name to Local Studio naming. |
